This part will teach you the basics of HubScript.

Template Structure

All projects created with HubScript are created based on the templates defined by the user, a template folder can have any name but must folow a certain structure, where each template folder must contain :

_template_

This folder will contain all the content that must be copied when creating a new project, things like the src folder, assests, configuration files not written in json, etc.

The structure of the content inside this folder will be structure of the final project created using this template.

template.config.json

This file gose along side the _template_ folder and inside of it will define the content of the configuration files written in json.

This file is important because HubScript automatically creates the package.json file for your project, so any change you want to be made automatically to your configuration files must be specified.

A simple template.config.json looks like this:

          {
            "filename.json" {
              "content1": true,
              "content2": "value",
              "content3": 100,
            },
            "filename2.json" {
              "content1": true,
              "content2": "value",
              "content3": 100,
            }
          }

Each key has the name of the file you want to change and the value of each key is the content of said file.

Exemple:

          {
            "package.json": {
              "description": "Created with HubScript.",
              "dependencies": {
                "@types/chalk": "^2.2.0",
                "@types/node": "^14.14.37",
                "chalk": "^4.1.1",
                "typescript": "^4.2.3"
              },
              "scripts": {
                "test": "echo \"Error: no test specified\" && exit 1",
                "start": "tsc -w"
              }
            },
            "tsconfig.json": {
              "compilerOptions": {
                "lib": ["ES2015", "DOM"],
                "module": "es2020",
                "baseUrl": "./src",
                "sourceMap": false,
                "target": "es6",
                "typeRoots": [
                  "./node_modules/@types/*",
                  "./src/@types/*"
                ]
              },
                "include": [
                  "src"
              ]
            }
          }
